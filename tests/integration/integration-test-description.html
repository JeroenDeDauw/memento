Integration Test Cases

MementoTest.test302StyleTimeNegotiationWholeProcess

Test Description: URI-R leads to URI-G which leads to URI-M. All headers in this process reflect the correct values.

Requirements Source

  • Original Code Implementation
  • Memento RFC - Sections 2.1.3, 3.1, 4.2.1
  • RFC 5988 - Section 6.2.2

Test Data File: time-negotiation-testdata.csv

Memento Mediawiki Extension Settings Required

  • default

Test Verification Steps

  1. URI-R response contains 200 status code
    • URI-R response contains Link header with timegate relation containing the correct the URI reference
  2. URI-G response contains 302 status code
    • URI-G entity does not exist
    • URI-G contains Location header with correct value for URI-M
    • URI-G contains Vary header with Accept-Datetime entry
    • URI-G contains Link header with the following relation tests
      • original latest-version exists and has correct value for the URI reference
      • timemap exists and has correct value for the URI reference
      • first last memento exists, has correct value for the URI reference, and contains datetime attribute (this relation exists if page only has one memento)
      • first memento exists, has correct value for the URI reference, and contains datetime attribute (this relation exists only if the page has more than one memento)
      • last memento exists, has correct value for the URI reference, and contains datetime attribute (this relation exists only if the page has more than one memento)
      • next successor-version memento exists, has correct value for the URI reference, and contains datetime attribute (this relation exists only if the page has more than one memento)
  3. URI-M response contains 200 status code
    • URI-M response contains Memento-Datetime header
    • URI-M response contains Link header with the following relation tests
      • original latest-version exists and has correct value for the URI reference
      • timemap exists and has correct value for the URI reference
      • first last memento exists, has correct value for the URI reference, and contains datetime attribute (this relation exists if page only has one memento)
      • first memento exists, has correct value for the URI reference, and contains datetime attribute (this relation exists only if the page has more than one memento)
      • last memento exists, has correct value for the URI reference', and contains datetime attribute (this relation exists only if the page has more than one memento)
      • next successor-version memento exists, has correct value for the URI reference, and contains datetime attribute (this relation exists only if the page has more than one memento)
      • timegate exists, has correct value for the URI reference, and contains datetime attribute
  4. URI-M entity contains no PHP errors

Note: prev predecessor-version memento needs to be incorporated into this test.

MementoTest.test200StyleTimeNegotiation

Test Description: Ensure 200-style time negotiation produces the correct headers

Requirements Source

  • Memento RFC - Section 4.2.1
  • RFC 5988 - Section 6.2.2
  • TODO

Test Data File: time-negotiation-testdata.csv

Memento Mediawiki Extension Settings Required $wgMementoTimeNegotiation = true

Test Verification Steps

  1. URI-R=URI-G generates 200 status code
  2. URI-R=URI-G contains Content-Location header with the correct URI-M value
  3. URI-R=URI-G contains Memento-Datetime header
  4. URI-R=URI-G contains Vary header with Accept-Datetime entry
  5. URI-R=URI-G contains Link header with the following relation tests
    • memento first exists, has the correct value for the the URI reference, and contains the 'datetime' attribute
    • memento last exists, has the correct value for the the URI reference, and contains the 'datetime' attribute
    • original timegate exists and has the correct value for the the URI reference

TimeMapTest.testGetTimeMap

Test Description: Correctly formatted URI-T is retrieved with the given URI

Requirements Source

  • Memento RFC - Sections 3.2, 5, 5.1.1

Test Data File: timemap-testdata.csv

Memento Mediawiki Extension Settings Required

  • default

Test Verification Steps

  1. URI-T response contains 200 status code
  2. URI-T response matches the contents of the expected timemap file
  3. URI-T response entity contains no PHP errors

Note: Future versions of this test will attempt to parse the incoming timemap to ensure it is in the correct format, but for now it just performs a simple string compare between the retrieved TimeMap and a file containing the expected contents.

Test Dataset Documentation

time-negotiation-testdata.csv

Github link: https://github.com/hariharshankar/mediawiki/blob/master/tests/data/wsdl-wiki/time-negotiation-testdata.csv

Fields:

  1. Identifier - Identifier for the test data entry
  2. Accept-Datetime - value passed in Accept-Datetime header for requests
  3. URI-R - original resource URI value
  4. original latest-version - expected value of the URI reference from the original latest-version relation in the Link header
  5. first memento - expected value of the URI reference from the first memento relation in the Link header
  6. last memento - expected value of the URI reference from the last memento relation in the Link header
  7. prev predecessor-version memento - expected value of the URI reference from the prev predecessor-version memento relation in the Link header
  8. next successor-version memento - expected value of the URI reference from the next successor-version memento relation in the Link header
  9. URI-M - expected URI value for the memento relation
  10. URI-G - expected URI value for the timegate relation (only used for tests with 302-style negotiation)
  11. Comment - comment on row in dataset indicating why it was chosen

memento-diffpage-testdata.csv
